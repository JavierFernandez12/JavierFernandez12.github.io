Project Management & Methodology

Challenge #1: Class Notebook & Git Commit Log

Git / Methodology / Version Control
1. Challenge Overview

This is a transversal challenge maintained throughout the entire course. It involves keeping a strict and structured Version Control history for all ASIR projects. The core requirement is to use specific semantic tags in every commit message: [CLASE] for work done during school hours and [CASA] for homework. This effectively turns the Git log into a "Class Notebook," documenting not just what changed, but where and when the effort was applied.

2. Key Complex Concepts Learned
  • Semantic Commits: I learned the importance of following a convention (like Conventional Commits). Writing "bugfix" or "update" is useless; writing [CASA] 'Fixed ASCII art alignment in Navarredonda script' provides context and traceability.
  • Atomic Commits: To make the log meaningful, I had to learn to commit small, logical units of work frequently, rather than doing one massive "mega-commit" at the end of the week.
3. Personal Reflection

At the beginning, it was tedious. I often forgot the tag and had to use git commit --amend to fix it. However, after a few weeks, it became a habit. Now, looking back at my git log gives me a feeling of satisfaction and order, seeing a clear timeline of my progress.

4. Proposed Improvements

To enforce this rule strictly, I would implement a Git Hook (pre-commit). This script would automatically reject any commit message that does not start with [CLASE] or [CASA], preventing human error and ensuring 100% compliance.

5. Technical Glossary
  • Commit: A snapshot of the project's changes in the repository at a specific point in time.
  • Staging Area: The intermediate area where changes are listed and reviewed before being committed.
  • Git Hook: Scripts that run automatically before or after Git events (like commit or push).
  • Amend: A Git operation used to modify the most recent commit (useful for fixing typos in messages).
  • Log: The record of all commits in the repository history.
6. Assignment Rating: 10/10
Reason: It builds professional discipline. In a real company, clean commit logs are essential for teamwork.

Challenge #2: Portfolio

Web Development / GitHub Pages / Documentation
1. Challenge Overview

The objective was to design and deploy a comprehensive web portal to showcase all the work done during the 2nd year of ASIR. The site had to cover all modules (Security, Databases, IAW, Python, Global), be written entirely in English, and follow a specific structure for each entry (Overview, Concepts, Feelings, Improvements, Glossary, Rating). The final requirement was to publish it globally using GitHub Pages.

2. Key Complex Concepts Learned
  • Static Site Architecture: I learned how to structure a multi-page static site without a backend, using relative linking strategies (`../css/style.css`) to maintain a consistent design across different directory levels.
  • Documentation as Code: Treating documentation not as a separate chore but as part of the codebase. Learning to synthesize complex technical tasks into clear, professional English summaries.
3. Personal Reflection

Developing this portfolio made me feel proud. Usually, we finish a practice and forget about it. Putting it all together makes me realize how much I have learned this year. Translating technical terms to English was challenging but necessary for the international market.

4. Proposed Improvements

Currently, the site uses static HTML files, which involves copy-pasting the menu on every page. To improve maintainability, I would switch to a Static Site Generator (SSG) like Jekyll or Hugo, or use JavaScript to load the header/footer dynamically. I would also add a "Dark Mode" toggle.

5. Technical Glossary
  • GitHub Pages: A static site hosting service that takes HTML, CSS, and JavaScript files straight from a repository.
  • Frontend: The part of a website that user interacts with directly (HTML/CSS).
  • Responsive Design: An approach to web design that makes web pages render well on a variety of devices and window or screen sizes.
  • Hyperlink: A reference to data that the user can follow by clicking or tapping.
  • Hosting: The service of providing storage and access for websites.
6. Assignment Rating & Feedback
Rating: 10/10
Reason: This is the most useful assignment for my career. It serves as a real CV to show to future employers. Improvement for the statement: The requirements were clear, but maybe adding a requirement to include a "Contact" form or LinkedIn integration would make it even more professional.
