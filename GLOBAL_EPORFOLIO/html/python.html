Python Programming Challenges

Challenge #1: Navarredonda Park Simulator

Logic / Conditionals / ASCII Art
1. Challenge Overview

I developed a Python program to calculate the entrance fee for an amusement park based on 7 input parameters (Age, Residence, Height, etc.) and 12 prioritized rules. The program had to evaluate complex conditions (e.g., "School group on Monday OR Friday") to apply discounts or surcharges to the base price (€45). The final output had to display the price using ASCII Art digits.

2. Key Complex Concepts Learned
  • Boolean Logic & Priority: I mastered the use of `if-elif-else` chains combined with `and/or` operators. The key challenge was respecting the priority table: rule #1 (Free for seniors > 65) had to be checked before rule #12 (Base Tariff), requiring a strictly ordered logic flow.
  • ASCII Art Output: Instead of a simple print, I learned to map integer results to multi-line string representations (ASCII numbers) to create a visual ticket.
3. Personal Reflection

I felt challenged by the logic. Debugging why a "Club Member from Madrid" wasn't getting the correct discount because a previous condition triggered first was tricky. Once the logic flowed correctly through all 12 rules, I felt accomplished.

4. Proposed Improvements

The code has many nested `if` statements. I would improve it by implementing a Rule Engine pattern: creating a list of Rule objects with 'condition' and 'action' attributes, and iterating through them. This would make adding a 13th rule much easier.

5. Technical Glossary
  • Boolean Operator: Symbols (AND, OR, NOT) used to connect and define the relationship between values in a conditional statement.
  • Float: A data type used to represent numbers with decimal points (e.g., Height 1.55).
  • Control Flow: The order in which individual statements, instructions, or function calls are executed or evaluated.
  • Hardcoding: Embedding data directly into the source code (e.g., the €45 base price) instead of using variables or config files.
  • Input Validation: The process of ensuring that user input is clean, correct, and useful (e.g., ensuring Age is an integer).
6. Assignment Rating & Feedback
Rating: 9.5/10
Reason: Excellent logic exercise. It forces you to think like a computer processing ordered rules. Improvement for the statement: The priority rules were clear, but the ASCII requirement was a sudden jump in difficulty compared to the logic part.

Challenge #2: Creative Graphics with Turtle

Python / Turtle / Algorithms
1. Challenge Overview

This assignment explored the Turtle library to understand coordinate systems and algorithmic drawing. The work consisted of three parts: 1. The Star: Drawing the specific "La Salle" star logo using precise angles. 2. The Name: Writing my name ("Root") on the canvas without using the write() function, meaning I had to program the path for every letter manually. 3. Creative Drawing: A complex freestyle design executing over 30 functions, where I used loops to create a spiral geometric pattern.

2. Key Complex Concepts Learned
  • Algorithmic Thinking: I learned to break down complex shapes (like a letter 'R') into small, reusable steps (forward, turn, pen up, pen down). It changed how I view vector graphics.
  • Loops for Geometry: Understanding how `for` loops combined with mathematical increments (e.g., `forward(i)` inside a loop) can create organic, spiral shapes that grow naturally.
3. Personal Reflection

Writing my name letter by letter was tedious but rewarding. Calculating the exact coordinates so the letters wouldn't overlap was a puzzle. However, seeing the final creative spiral animate on the screen was mesmerizing and made me feel artistic using code.

4. Proposed Improvements

My code is currently linear. To improve it, I would define a function for each letter (e.g., `draw_R()`, `draw_O()`) to make the code modular and reusable for writing other words.

5. Technical Glossary
  • Turtle: A pre-installed Python library that enables users to create pictures and shapes by providing a virtual canvas.
  • Canvas: The window or area where the drawing output is displayed.
  • Loop (Iteration): A sequence of instructions that is continually repeated until a certain condition is reached.
  • Coordinates (x,y): A set of values that show an exact position on the canvas grid.
  • Refactoring: The process of restructuring existing computer code without changing its external behavior (e.g., creating functions).
6. Assignment Rating & Feedback
Rating: 8/10
Reason: A fun way to learn syntax, though it has limited real-world application for sysadmins compared to scripting.
