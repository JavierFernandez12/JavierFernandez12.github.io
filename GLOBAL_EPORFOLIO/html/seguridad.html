Challenge #1: attack_simulation

SAD / Security
1. Challenge Overview

In this task, I worked with dictionary generation tools to prepare for brute-force attacks. I used Pydictor to create two specific wordlists: one with numeric patterns and another with uppercase letters. Then, I used Dymerge to fuse these two lists into a single master dictionary named diccionarion.

2. Key Complex Concepts Learned
  • Dictionary Merging: I learned how to combine multiple small, specific wordlists into a larger, more effective payload using tools like Dymerge, rather than relying on static lists.
  • Pattern-based Generation: I understood how to use Pydictor to generate passwords based on specific rules (length, character type) instead of random guessing.
3. Personal Reflection

At first, I felt a bit confused by the command-line flags of Pydictor. However, seeing Dymerge successfully combine the files gave me a sense of satisfaction. It felt like preparing the "ammunition" before the battle.

4. Proposed Improvements

The process was manual. To improve it, I would write a small Bash script that automatically installs the tools if they are missing and generates the dictionary with a single command to save time.

5. Technical Glossary
  • Brute-force: A trial-and-error method used to decode encrypted data or passwords.
  • Wordlist: A text file containing a list of potential passwords used for attacks.
  • Base64: A binary-to-text encoding scheme that represents binary data in an ASCII string format.
  • Merge: The operation of combining two or more files or datasets into one.
  • Pydictor:A powerful dictionary builder tool written in Python.
6. Assignment Rating: 8/10
Feedback: Good introduction to the tools, but I would have liked to try generating a dictionary based on a specific person's social media data (profiling).
